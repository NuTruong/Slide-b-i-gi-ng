# Contribution Guidelines

Welcome to the Slide-b-i-gi-ng repository! We appreciate your interest in contributing to this project. Please follow the guidelines below to ensure a smooth contribution experience.

## Forking and Cloning the Repository
1. **Fork the Repository**: Visit the Slide-b-i-gi-ng GitHub page and click on the `Fork` button in the upper right corner to create your copy of the repository.
2. **Clone Your Fork**: Clone your forked repository to your local machine using the following command:
   ```bash
   git clone https://github.com/your-username/Slide-b-i-gi-ng.git
   ```
   Replace `your-username` with your GitHub username.

## Branch Naming Conventions
- Use descriptive names for branches to indicate the purpose.
- Format: `feature/your-feature-name` for new features, `bugfix/your-bugfix-name` for bug fixes.

Example:
```bash
git checkout -b feature/add-new-slider
```

## Code Style Guidelines
- Follow the existing coding style and conventions used in this repository.
- Ensure your code is properly indented and commented.
- Run any relevant formatting tools before submitting your changes.

## Commit Message Format
- Your commit message should be clear and descriptive.
- Format: `type: subject` where type can be `feat`, `fix`, `docs`, or `style`.

Example:
```
feat: add new sliding feature
```

## Pull Request Process
1. **Create a Pull Request**: After pushing your changes to your fork, navigate to the original repository and click on `New Pull Request`.
2. **Fill Out the Pull Request Template**: Provide a clear description of the changes and link any related issues.
3. **Review Process**: All pull requests will be reviewed, and feedback may be provided. Be ready to make changes based on reviewer suggestions.

## Bug Report Templates
If you find a bug, please report it using the following template:
```markdown
### Bug Description
A clear and concise description of what the bug is.

### Steps to Reproduce
1. Step 1
2. Step 2
3. Step 3

### Expected Behavior
A clear description of what you expected to happen.

### Actual Behavior
A clear description of what actually happened.
```  

## Community Guidelines
- Be respectful and considerate to all community members.
- Avoid personal attacks and inflammatory language.
- Help each other by providing constructive criticism.

Thank you for contributing to Slide-b-i-gi-ng! We look forward to your contributions!
